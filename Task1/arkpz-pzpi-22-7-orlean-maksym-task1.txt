1	МЕТА РОБОТИ

Метою даної лабораторної роботи є розробка документа Vision and Scope для програмної системи моніторингу дорожнього руху. Система призначена для збору та аналізу даних про транспортний потік, виявлення заторів та небезпечних ділянок, а також підвищення ефективності дорожніх систем. Головні завдання включають інтеграцію IoT-пристроїв, створення серверної частини з REST API та розробку веб-та мобільних клієнтів для візуалізації та аналізу даних.

2	ХІД РОБОТИ

У рамках лабораторної роботи було проведено аналіз вимог до програмної системи та визначено її ключові функціональні аспекти. Головна увага була приділена створенню інтеграції IoT-пристроїв для збору та обробки даних про дорожній рух.
Система базується на інтеграції камер з розпізнаванням транспортних засобів та датчиків, що вимірюють щільність та швидкість руху. Зібрані дані оброблюються серверною частиною, реалізованою на базі Python та фреймворку Flask, з записом у базу даних PostgreSQL.
Проведено дослідження існуючих рішень на ринку, таких як Google Maps та Waze. Аналіз дозволив визначити недоліки аналогів, такі як обмеження у точності аналізу та відсутність інтеграції з IoT-даними. Унікальні переваги нашої системи включають реальний моніторинг дорожнього руху та забезпечення аналітичних рекомендацій щодо оптимізації дорожньої мережі.
На першому етапі реалізовано серверну частину системи, яка підготовлена до інтеграції з IoT-пристроями. Веб-інтерфейс та мобільний додаток планується до розробки в наступних етапах.

3	ВИСНОВКИ

У результаті виконаної роботи було створено концепцію програмної системи моніторингу дорожнього руху, що базується на сучасних технологіях IoT та серверного програмного забезпечення. Головним досягненням є розробка серверної частини, яка забезпечує обробку даних з камер і датчиків дорожнього руху та їх зберігання в базі даних. Це є критичним компонентом системи, оскільки саме він забезпечує її функціональність та масштабованість.
Проведений аналіз аналогів підтвердив, що існуючі рішення на ринку не повністю задовольняють потреби користувачів у зручності та точності моніторингу. Це створює значний потенціал для впровадження нашої системи, яка вирізняється інтеграцією IoT-пристроїв для збору даних у реальному часі, що дозволяє швидко виявляти затори, аварії та інші проблеми на дорогах.
Подальші етапи роботи передбачають розробку веб- та мобільних інтерфейсів, які забезпечать зручний доступ до функцій системи для кінцевих користувачів. Реалізація цих компонентів дозволить створити повноцінний продукт, готовий до впровадження у реальних умовах для покращення дорожньої інфраструктури та зручності водіїв.

1	BUSINESS REQUIREMENTS (БІЗНЕС-ВИМОГИ)

1.1	Background (Передумови)

RoadSense — це інноваційна система, призначена для моніторингу дорожнього руху з використанням сучасних IoT-технологій. Основною метою проєкту є забезпечення точного збору даних про стан трафіку, виявлення заторів, небезпечних ділянок і потенційних аварій. Система допомагає містам, компаніям та інфраструктурним управлінням ефективно організовувати дорожній рух, знижувати час простоїв та підвищувати загальну безпеку.
Впровадження RoadSense передбачає інтеграцію камер із функцією розпізнавання транспортних засобів, датчиків трафіку та інших IoT-пристроїв, які в реальному часі передають інформацію на сервер. Кінцеві користувачі можуть переглядати детальну інформацію через зручний веб-інтерфейс або мобільний додаток, отримуючи доступ до таких функцій, як перегляд завантаженості доріг, отримання попереджень про аварії та рекомендації з оптимізації маршрутів.

1.2	(Business Opprtunity) Бізнес можливості

RoadSense надає містам і підприємствам потужний інструмент для управління транспортними потоками та підвищення безпеки на дорогах. Використання IoT-пристроїв для збору даних дозволяє:
•	Аналізувати поточний стан трафіку в реальному часі.
•	Визначати проблемні ділянки, такі як аварії чи затори.
•	Оптимізувати маршрути транспорту, скорочуючи час поїздок і витрати на пальне.
Основні переваги системи включають:
•	Інтеграцію IoT: датчики трафіку та камери забезпечують повний моніторинг дорожнього руху.
•	Реальний час: актуальна інформація про стан доріг, доступна 24/7.
•	Простота використання: інтуїтивно зрозумілий веб-інтерфейс і мобільний додаток для користувачів різного рівня технічної підготовки.
•	Масштабованість: можливість адаптації до різних обсягів трафіку та територій.
Водночас існують певні виклики:
•	Залежність від стабільного інтернет-з’єднання для передачі даних.
•	Складнощі інтеграції з існуючими системами в деяких містах.
У порівнянні з аналогами, такими як Wialon або Geotab, RoadSense пропонує комплексне рішення, яке включає моніторинг трафіку, аналіз заторів і рекомендації з оптимізації руху в межах однієї платформи. Це робить його ідеальним вибором для інфраструктурного управління та організації дорожнього руху.

1.3	Способи монетизації

•	Підписка для користувачів:
Система RoadSense може надавати базовий безкоштовний функціонал для моніторингу трафіку в реальному часі. Для користувачів, які потребують розширених можливостей, передбачений преміум-план. Цей план включатиме доступ до історичних даних, прогнозування заторів, персоналізовані сповіщення, автоматизовані звіти, а також аналітичні інструменти для визначення найбільш ефективних маршрутів.
•	Партнерства з муніципалітетами та транспортними компаніями:
Надання інтегрованих рішень для міських управлінь транспорту, транспортних компаній і служб доставки. Партнерські програми можуть включати створення кастомізованих функцій для конкретних клієнтів, наприклад, детальний аналіз трафіку для міського планування або спеціальні API для логістичних компаній.
•	Реклама:
У веб-інтерфейсі та мобільному додатку можна розміщувати контекстну рекламу партнерів, таких як постачальники IoT-обладнання, розробники програмного забезпечення для транспорту, страхові компанії, виробники транспортних засобів і сервісних центрів.
•	Консультаційні послуги:
Надання послуг із впровадження системи у великих інфраструктурних проєктах, проведення навчання персоналу користувачів і розробки індивідуальних рішень на замовлення.
•	Додаткові послуги:
Пропонування клієнтам розширених функцій, таких як:
•	Деталізована аналітика щодо завантаженості доріг.
•	API-доступ для інтеграції з іншими бізнес-процесами.
•	Автоматизовані прогнози щодо стану доріг і транспортних потоків.
•	Ліцензування даних:
Продавати доступ до знеособлених даних про трафік великим корпораціям, аналітичним агентствам або урядовим організаціям для проведення досліджень і планування.

2	VISION OF THE SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)

2.1	Vision Statement (Окреслення концепції)

Програмна система RoadSense є інтелектуальним рішенням для моніторингу та аналізу дорожнього руху в режимі реального часу. Вона інтегрує IoT-пристрої (камери, датчики трафіку) для збору даних про стан доріг, завантаженість, швидкість руху транспортних засобів та виявлення аварійних ситуацій. Система дозволяє оперативно оцінювати стан дорожнього руху, прогнозувати затори та оптимізувати маршрути.
Кінцеві користувачі (транспортні департаменти, логістичні компанії, приватні водії) отримують доступ до даних через веб-інтерфейс і мобільний додаток. Це сприяє скороченню затримок, підвищенню безпеки на дорогах та зниженню витрат на перевезення.

2.2	Major Features (Головна функціональність)

Основні функції системи:
MF-1) Моніторинг стану дорожнього руху в реальному часі:
Інтеграція з камерами та датчиками для відображення завантаженості доріг на карті в реальному часі.
MF-2) Прогнозування заторів та виявлення аварій:
Аналіз даних про швидкість руху транспортних засобів і щільність трафіку для ідентифікації можливих проблемних ділянок.
MF-3) Збір та зберігання історичних даних:
Збереження даних про дорожній рух для проведення довгострокових аналізів, створення звітів та розробки транспортних стратегій.
MF-4) Оповіщення користувачів:
Надсилання push-сповіщень про затори, аварії або інші зміни на визначених маршрутах через мобільний додаток.
MF-5) Інтерактивне планування маршрутів:
Автоматична генерація альтернативних маршрутів на основі даних про поточний стан доріг.
MF-6) Інтеграція з існуючими системами:
Підтримка API для синхронізації з логістичними платформами, системами управління транспортом та іншими програмними рішеннями.

3	SCOPE AND LIMITATIONS (РАМКИ ТА ОБМЕЖЕННЯ ПРОЄКТУ)

3.1	Scope of Initial Release (Рамки первинного випуску)

Функціональні можливості, заплановані для початкового релізу програмної системи RoadSense:

•	MF 1-2) Виявлення заторів, аварій і небезпечних ділянок з автоматичним повідомленням про них користувачів.
•	MF 2-1) Можливість налаштування параметрів моніторингу для окремих ділянок дороги та маршрутів.
•	MF 3-1) Інтеграція з основними системами GPS-трекінгу для синхронізації даних про транспортні засоби.

•	MF 2-2) Додавання підтримки додаткових типів датчиків, таких як акустичні сенсори, для покращення точності моніторингу.
•	MF 3-3) Інтеграція функцій автоматичного створення альтернативних маршрутів з урахуванням завантаженості доріг та погодних умов.
•	MF 4-2) Покращення персоналізації даних для різних користувачів (наприклад, логістичних компаній, муніципальних служб).
•	MF 5-2) Оптимізація адміністрування з додаванням функцій автоматичного оновлення та синхронізації даних між користувачами.
•	MF 6-2) Локалізація інтерфейсу для різних мов і регіонів, що дозволить розширити географію використання системи.

4	BUSINESS CONTEXT (БІЗНЕС-КОНТЕКСТ)

4.3	Operating Enviroment (Робоче середовище)

Система RoadSense орієнтована на використання в різних регіонах і часових зонах, забезпечуючи безперервний доступ до даних про дорожню ситуацію, технічний стан транспортних засобів та історію їх використання. Основна мета — надання точних і своєчасних даних для прийняття ефективних рішень з управління транспортом і логістикою.
Користувачі отримають можливість:
•	Переглядати в реальному часі місцезнаходження транспортних засобів.
•	Аналізувати стан технічного оснащення (двигун, температура, шини).
•	Отримувати автоматизовані сповіщення про необхідність обслуговування чи виявлені проблеми.
Система інтегрується з різноманітними пристроями, включаючи GPS-трекери та датчики IoT, для збору та передачі даних. Для зручності користувачів надається веб-платформа та мобільний додаток. Стабільне інтернет-з'єднання є ключовим для безперебійної роботи системи, зокрема для надсилання актуальних даних і рекомендацій.
Мови програмування:
•	JavaScript (Node.js, Express): Для створення серверної частини з RESTful API, інтеграції з базою даних і обробки запитів.
•	JavaScript (React): Для розробки інтерактивного веб-інтерфейсу з багатим функціоналом.
Інфраструктура серверної частини:
•	Express: Легковаговий веб-фреймворк для створення RESTful API та обробки запитів від клієнтів.
•	MongoDB: NoSQL база даних для зберігання інформації про транспортні засоби, користувачів та їхні дії.
•	Mongoose: Бібліотека для роботи з MongoDB, яка спрощує взаємодію з базою даних.
•	MQTT: Протокол для передачі даних між IoT пристроями (GPS-трекерами, сенсорами) і сервером у реальному часі.
•	JWT (JSON Web Tokens): Технологія авторизації для захисту персональних даних і забезпечення безпеки доступу.
•	Docker: Для контейнеризації серверного оточення, що полегшує процеси розробки, тестування та деплойменту.
Інфраструктура фронтенд частини:
•	React: Бібліотека для створення сучасного, зручного веб-інтерфейсу.
•	Material-UI: Інструмент для стилізації інтерфейсу, який забезпечує інтуїтивний і привабливий дизайн.
•	Axios: Бібліотека для виконання HTTP-запитів до API, забезпечуючи швидку взаємодію з сервером.
Інфраструктура мобільного додатку:
•	Kotlin/Java (Android): Для розробки мобільного додатку, який забезпечує збір і передачу даних від IoT пристроїв на сервер.
•	SQLite: Для тимчасового зберігання даних у разі відсутності інтернет-з'єднання.
•	Firebase Cloud Messaging: Для надсилання push-сповіщень користувачам.
Система буде оптимізована для масштабування та адаптована до потреб як локальних, так і міжнародних користувачів, забезпечуючи високу продуктивність і стабільність роботи.
